# iSearch 增强功能文档

本文档描述了对照 Linux FSearch 功能增强后的 iSearch 模块新特性。

## 🔍 高级搜索语法

### 基本搜索
- `关键词` - 基本搜索
- `"精确短语"` - 精确匹配短语
- `+必须词` - 必须包含该关键词
- `-排除词` - 排除包含该关键词的结果

### 逻辑操作符
- `AND` 或 `&&` - 逻辑与
- `OR` 或 `||` - 逻辑或  
- `NOT` 或 `!` - 逻辑非
- `( )` - 分组

### 文件筛选
- `filetype:pdf` - 按文件类型筛选
- `filename:report` - 按文件名筛选
- `ext:pdf;doc;txt` - 按扩展名筛选（支持多个）
- `path:/home/user` - 按路径筛选

### 大小筛选
- `size:>1MB` - 大于1MB的文件
- `size:<=500KB` - 小于等于500KB的文件
- `size:1GB..2GB` - 1GB到2GB之间的文件
- 支持单位：B, KB, MB, GB, TB

### 日期筛选
- `date:today` - 今天修改的文件
- `date:yesterday` - 昨天修改的文件
- `date:>2023-01-01` - 2023年1月1日后修改的文件
- `date:2023` - 2023年修改的文件

### 修饰符
- `case:关键词` - 区分大小写
- `exact:关键词` - 精确匹配
- `file:关键词` - 仅搜索文件
- `folder:关键词` - 仅搜索文件夹
- `regex:"模式"` - 正则表达式搜索

### 示例查询
```
# 基本组合搜索
project +important filetype:pdf

# 复杂逻辑搜索
"quarterly report" AND size:>1MB

# 多条件筛选
(document OR report) ext:pdf;doc date:>2023-01-01

# 精确匹配和排除
case:exact:"Project Name" -draft
```

## 📊 性能指标

- 平均搜索时间：< 100ms（小型索引）

- 监控缓冲：< 10MB
- 总内存占用：< 200MB

## 🔧 故障排除

### 常见问题
1. **搜索结果不准确** - 检查高级搜索语法是否正确
