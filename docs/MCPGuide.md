# MCP集成使用指南

## 📖 概述

MCP (Model Context Protocol) 是一个开放标准，允许AI助手与外部工具和资源进行安全、标准化的交互。SeeU Desktop 完整实现了MCP协议，为AI助手提供了强大的工具调用能力。

## 🚀 快速开始

### 1. 添加MCP服务器

#### 使用表单输入（推荐新手）
1. 打开 iTools 模块
2. 切换到"MCP服务器"标签
3. 点击"添加服务器"按钮
4. 选择"表单输入"模式
5. 填写服务器信息：
   - **服务器名称**: 给服务器起一个易识别的名称
   - **描述**: 简要描述服务器功能
   - **传输类型**: 选择 stdio 或 SSE
   - **命令**: 服务器启动命令
   - **参数**: 命令行参数（每行一个）
   - **工作目录**: 服务器运行目录
   - **环境变量**: 必要的环境变量

#### 使用JSON输入（推荐高级用户）
1. 选择"JSON输入"模式
2. 输入标准MCP服务器配置JSON：
```json
{
  "mcpServers": {
    "my-server": {
      "command": "node",
      "args": ["path/to/server.js"],
      "env": {
        "API_KEY": "your-api-key"
      }
    }
  }
}
```

### 2. 测试服务器
1. 添加服务器后，状态显示为 🔴（红灯）
2. 点击"🧪 测试"按钮
3. 系统会验证服务器连接和功能
4. 测试通过后状态变为 🟢（绿灯）

### 3. 在AI助手中使用
1. 打开AI助手
2. 在MCP服务器下拉菜单中选择已测试的服务器
3. 发送需要工具协助的消息
4. AI助手会自动识别并调用相应工具
5. 确认工具调用后查看执行结果

## 🛠 预设服务器模板

### 开发工具类

#### Everything Server
- **功能**: 快速文件搜索
- **用途**: 在大型项目中快速定位文件
- **配置**: 需要安装Everything搜索引擎

#### Git Server
- **功能**: Git版本控制操作
- **用途**: 查看提交历史、分支状态、文件变更
- **配置**: 需要Git命令行工具

### 文件系统类

#### File System Server
- **功能**: 文件和目录操作
- **用途**: 读取、写入、创建、删除文件
- **配置**: 通常无需额外配置

### 数据库类

#### SQLite Server
- **功能**: SQLite数据库查询
- **用途**: 查询和操作SQLite数据库
- **配置**: 需要指定数据库文件路径

### Web服务类

#### Fetch Server
- **功能**: HTTP请求处理
- **用途**: 获取网页内容、API调用
- **配置**: 可能需要代理设置

#### Brave Search Server
- **功能**: 网络搜索
- **用途**: 获取最新信息、研究资料
- **配置**: 需要Brave Search API密钥

### 生产力工具类

#### Memory Server
- **功能**: 临时数据存储
- **用途**: 在对话中保存和检索信息
- **配置**: 通常无需额外配置

## 🔧 高级配置

### 服务器状态管理

#### 状态说明
- **🔴 红灯**: 新添加或配置修改的服务器
- **🟡 黄灯**: 连接成功但未测试功能的服务器
- **🟢 绿灯**: 功能测试通过，可用于AI助手的服务器
- **❌ 错误**: 连接或测试失败的服务器

#### 批量操作
- **批量启动**: 同时启动多个服务器
- **批量停止**: 同时停止多个服务器
- **批量测试**: 同时测试多个服务器功能
- **批量删除**: 同时删除多个服务器配置

### 性能监控

#### 实时监控
- **连接状态**: 实时显示服务器连接状态
- **响应时间**: 监控服务器响应延迟
- **资源使用**: 跟踪CPU和内存使用情况
- **错误率**: 统计请求成功率和错误率

#### 健康检查
- **自动检测**: 定期检查服务器健康状态
- **故障恢复**: 自动重连断开的服务器
- **性能警告**: 当性能指标异常时发出警告

## 🤖 AI助手工具调用

### 工具调用流程

#### 1. 工具识别
AI助手分析用户消息，识别需要调用的工具类型：
- 文件操作需求 → File System Server
- 搜索需求 → Everything Server 或 Brave Search Server
- 数据查询需求 → SQLite Server
- 网络请求需求 → Fetch Server

#### 2. 工具调用请求
AI助手生成工具调用请求，包含：
- 工具名称和服务器信息
- 调用参数和配置
- 预期结果格式

#### 3. 用户确认
系统显示工具调用确认对话框：
- 显示将要执行的操作
- 展示调用参数
- 提供执行/取消选项

#### 4. 工具执行
用户确认后系统执行工具调用：
- 与MCP服务器通信
- 传递参数并获取结果
- 处理错误和异常情况

#### 5. 结果反馈
工具执行完成后：
- 显示执行结果
- 自动发送结果给AI获取智能反馈
- 保存执行历史记录

### 安全机制

#### 用户确认
- 所有工具调用都需要用户手动确认
- 显示详细的操作信息和参数
- 用户可以在执行前取消操作

#### 权限控制
- 服务器运行在受限环境中
- 限制文件系统访问范围
- 禁止执行危险操作

#### 日志记录
- 记录所有工具调用和结果
- 保存详细的执行日志
- 便于问题排查和审计

## 🔍 故障排除

### 常见问题

#### 服务器无法启动
1. 检查命令路径是否正确
2. 验证参数格式是否正确
3. 确认工作目录是否存在
4. 检查环境变量是否设置

#### 连接超时
1. 检查网络连接
2. 验证服务器是否正在运行
3. 调整超时设置
4. 查看防火墙设置

#### 工具调用失败
1. 检查服务器状态是否为绿灯
2. 验证工具参数是否正确
3. 查看详细错误日志
4. 重新测试服务器功能

### 调试技巧

#### 查看日志
- 在设置中启用详细日志
- 查看MCP通信日志
- 分析错误堆栈信息

#### 测试连接
- 使用测试功能验证服务器
- 检查服务器能力信息
- 验证工具列表是否正确

#### 性能分析
- 监控服务器响应时间
- 检查资源使用情况
- 分析性能瓶颈

## 📚 最佳实践

### 服务器配置
1. **命名规范**: 使用清晰、描述性的服务器名称
2. **分类管理**: 按功能类型组织服务器
3. **环境隔离**: 为不同环境配置不同的服务器
4. **定期测试**: 定期测试服务器功能确保正常运行

### 工具使用
1. **明确需求**: 在请求中明确说明需要什么工具
2. **参数准确**: 提供准确的参数和配置信息
3. **结果验证**: 检查工具执行结果是否符合预期
4. **错误处理**: 妥善处理工具调用失败的情况

### 安全考虑
1. **最小权限**: 只给服务器必要的权限
2. **定期审计**: 定期检查服务器配置和使用情况
3. **敏感信息**: 避免在配置中暴露敏感信息
4. **访问控制**: 限制对重要资源的访问
