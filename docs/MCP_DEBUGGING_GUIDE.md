# MCP工具识别问题调试指南

## 问题描述

根据日志反馈，MCP服务器"simple-tool"的工具识别存在问题：
- 日志显示工具数量为0，但实际上该服务器应该有一个fetch工具
- 问题出现在能力提取或数据库存储阶段

## 调试步骤

### 第一步：检查MCP服务器连接状态

1. 启动应用程序并设置日志级别为INFO：
   ```bash
   RUST_LOG=info cargo run
   ```

2. 进入iTools -> MCP设置页面

3. 查找"simple-tool"服务器的状态信息：
   ```
   📊 服务器状态检查 - 'simple-tool' (uuid):
     - 连接状态: Connected/Disconnected
     - 健康状态: Healthy/Unhealthy
     - rmcp服务可用: true/false
   ```

### 第二步：检查能力提取过程

1. 在MCP设置中点击"simple-tool"服务器的测试按钮

2. 观察日志中的能力提取过程：
   ```
   🔍 开始查询服务器能力: simple-tool
   ✅ 使用现有rmcp服务提取能力 - 服务器: 'simple-tool'
   🔍 原始工具响应: {"tools": [...]}
   📊 成功解析 X 个工具从rmcp服务响应
   ```

3. 如果看到工具数量为0，检查原始响应格式

### 第三步：检查原始MCP响应

查找以下日志来检查MCP服务器的原始响应：
```
🔍 原始工具响应: {...}
📋 响应结构: {...}
```

可能的问题：
- MCP服务器没有返回tools数组
- tools数组为空
- 响应格式不符合预期

### 第四步：检查数据库存储

1. 查找数据库同步日志：
   ```
   🔧 找到服务器 'simple-tool' 的运行时能力: 工具=X, 资源=Y, 提示=Z
   ```

2. 如果运行时能力为空，检查是否有静态能力配置：
   ```
   🔧 使用服务器 'simple-tool' 的静态能力配置
   ```

### 第五步：手动触发能力提取（高级调试）

如果需要强制重新提取能力，可以：

1. 在代码中调用手动提取方法（需要开发环境）
2. 重新启动MCP服务器
3. 删除并重新添加MCP服务器配置

## 常见问题和解决方案

### 问题1：MCP服务器未正确启动
**症状**: 连接状态为Disconnected
**解决方案**: 
- 检查MCP服务器进程是否运行
- 验证命令和参数配置
- 检查端口是否被占用

### 问题2：MCP协议版本不兼容
**症状**: 连接成功但工具查询失败
**解决方案**:
- 检查MCP服务器支持的协议版本
- 更新rmcp客户端或MCP服务器

### 问题3：工具响应格式错误
**症状**: 原始响应中没有tools数组或格式不正确
**解决方案**:
- 检查MCP服务器的list_tools实现
- 验证响应是否符合MCP规范

### 问题4：能力解析失败
**症状**: 有原始响应但解析后工具数量为0
**解决方案**:
- 检查工具名称和描述字段
- 验证inputSchema格式
- 检查JSON解析逻辑
