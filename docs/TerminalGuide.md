# iTerminal 安全终端使用指南

## 📖 概述

iTerminal 是 SeeU Desktop 内置的安全终端模块，提供了一个受控的命令行环境。与传统终端不同，iTerminal 只允许执行预定义的安全命令，确保系统安全的同时提供必要的命令行功能。

## 🚀 快速开始

### 1. 打开终端
- 点击左侧导航栏的 🖥 图标
- 或使用快捷键进入终端模块

### 2. 基本操作
- 在输入框中输入命令
- 按 Enter 键执行命令
- 使用上下箭头键浏览命令历史
- 点击 "+" 按钮创建新标签页

### 3. 多标签页管理
- **创建标签页**: 点击 "+" 按钮
- **切换标签页**: 点击标签页标题
- **关闭标签页**: 点击标签页的 "×" 按钮
- **重命名标签页**: 右键点击标签页标题

## 📋 支持的安全命令

### 📁 文件和目录操作 (10个命令)

#### `ls` / `dir` - 目录列表
```bash
ls                    # 显示当前目录内容
ls -l                 # 详细列表格式
ls -a                 # 显示隐藏文件
ls -h                 # 人性化文件大小显示
ls /path/to/directory # 显示指定目录内容
```

#### `cd` - 目录切换
```bash
cd /path/to/directory # 切换到指定目录
cd ..                 # 返回上级目录
cd ~                  # 切换到用户主目录
cd                    # 切换到用户主目录
```

#### `pwd` - 当前目录
```bash
pwd                   # 显示当前工作目录的完整路径
```

#### `cat` / `type` - 文件内容显示
```bash
cat filename.txt      # 显示文件全部内容
type filename.txt     # Windows风格的文件内容显示
```

#### `head` / `tail` - 文件头尾显示
```bash
head filename.txt     # 显示文件前10行
head -n 20 file.txt   # 显示文件前20行
tail filename.txt     # 显示文件后10行
tail -n 5 file.txt    # 显示文件后5行
```

#### `wc` - 文件统计
```bash
wc filename.txt       # 显示行数、单词数、字符数
wc -l filename.txt    # 只显示行数
wc -w filename.txt    # 只显示单词数
wc -c filename.txt    # 只显示字符数
```

#### `find` - 文件查找
```bash
find . -name "*.txt"  # 查找当前目录下的txt文件
find /path -name file # 在指定路径查找文件
```

#### `tree` - 目录树显示
```bash
tree                  # 显示当前目录的树形结构
tree /path            # 显示指定目录的树形结构
```

#### `du` - 磁盘使用情况
```bash
du                    # 显示目录大小
du -h                 # 人性化显示大小
```

### 🔍 搜索和过滤 (2个命令)

#### `grep` - 文本搜索
```bash

grep -i "pattern" file.txt     # 忽略大小写搜索
grep -n "pattern" file.txt     # 显示行号
grep -r "pattern" directory/   # 递归搜索目录
```

#### `which` - 命令位置查找
```bash
which ls              # 查找ls命令的位置
which python          # 查找python命令的位置
```

### 命令白名单
- 只允许执行预定义的23个安全命令
- 禁止执行系统级危险操作

- 对命令参数进行安全检查
- 防止路径遍历攻击

- 命令在受限环境中执行

## 🌐 远程服务器管理

### 📖 功能概述

iTerminal 内置了强大的远程服务器管理功能，让您可以轻松管理多台远程服务器并快速建立SSH连接。这个功能特别适合需要频繁连接多台服务器的开发者、运维人员和系统管理员。

### 🚀 快速开始

#### 1. 访问远程服务器管理
- **方式一**: 点击左侧导航栏的 🖥 图标进入终端，然后点击设置中的"🖥️ 管理远程服务器"
- **方式二**: 在终端设置页面直接点击"➕ 添加服务器"按钮

#### 2. 添加第一台服务器
1. 点击"➕ 添加服务器"按钮
2. 填写基本信息（名称、主机、用户名）
3. 选择认证方式（推荐使用SSH Agent或私钥）
4. 点击"保存"完成添加
5. 点击"🔗 连接"测试连接

### 🔧 详细配置指南

#### 服务器基本信息

##### 必填字段
- **名称**: 服务器的显示名称，建议使用有意义的名称如"生产Web服务器"
- **主机**: 服务器的IP地址或域名，如 `192.168.1.100` 或 `server.example.com`
- **用户名**: SSH登录用户名，如 `root`、`ubuntu`、`admin` 等

##### 可选字段
- **端口**: SSH服务端口，默认为22，如果服务器使用非标准端口请修改
- **工作目录**: 连接后自动切换到的目录，如 `/home/user/projects`
- **描述**: 服务器的详细描述，便于识别用途
- **标签**: 用于分类管理，如 `生产`、`测试`、`开发` 等

#### 认证方式配置

##### 1. SSH Agent认证（推荐）
```bash
# 适用场景：已配置SSH Agent的用户
# 优点：最安全，无需存储密码
# 前提：系统中已运行SSH Agent并加载了密钥
```

**配置步骤**:
1. 选择"SSH Agent"认证方式
2. 确保本地SSH Agent正在运行
3. 确保目标服务器的公钥已添加到 `~/.ssh/authorized_keys`

**验证SSH Agent**:
```bash
# 检查SSH Agent是否运行
ssh-add -l

# 如果没有运行，启动SSH Agent
eval "$(ssh-agent -s)"

# 添加私钥到SSH Agent
ssh-add ~/.ssh/id_rsa
```

##### 2. 私钥文件认证
```bash
# 适用场景：有私钥文件的用户
# 优点：安全性高，支持密钥密码保护
# 注意：私钥文件权限需要设置为600
```

**配置步骤**:
1. 选择"私钥"认证方式
2. 点击"浏览"选择私钥文件（通常在 `~/.ssh/` 目录下）
3. 如果私钥有密码保护，输入私钥密码
4. 私钥密码会被加密存储

**常见私钥文件**:
- `~/.ssh/id_rsa` - RSA私钥
- `~/.ssh/id_ed25519` - Ed25519私钥
- `~/.ssh/id_ecdsa` - ECDSA私钥

##### 3. 密码认证
```bash
# 适用场景：只能使用密码登录的服务器
# 注意：密码会被加密存储，但建议使用密钥认证
```

**配置步骤**:
1. 选择"密码"认证方式
2. 输入登录密码
3. 密码会使用AES-256-GCM加密存储

### 🔒 安全特性

#### 加密存储
- **算法**: AES-256-GCM加密算法
- **密钥管理**: 加密密钥存储在系统密钥环中
- **数据保护**: 密码和私钥密码都经过加密处理
- **访问控制**: 只有当前用户可以访问加密数据

#### 系统密钥环集成
- **Windows**: 使用Windows Credential Manager
- **macOS**: 使用Keychain Access
- **Linux**: 使用Secret Service API (gnome-keyring等)

#### 配置文件安全
```json
{
  "version": "1.0",
  "servers": [
    {
      "id": "uuid",
      "name": "服务器名称",
      "host": "server.example.com",
      "auth_type": "password",
      "encrypted_data": {
        "password": {
          "data": "base64_encrypted_data",
          "nonce": "base64_nonce"
        }
      }
    }
  ]
}
```

### 🔧 连接管理

#### 建立连接
1. 在服务器列表中找到目标服务器
2. 点击"🔗 连接"按钮
3. 系统自动生成SSH命令并创建新会话
4. 连接成功后显示远程服务器的Shell提示符

#### SSH命令生成
```bash
# 系统会根据配置自动生成类似以下的SSH命令：

# SSH Agent认证
ssh -o ServerAliveInterval=60 -o ServerAliveCountMax=3 -o ConnectTimeout=10 user@host

# 私钥认证
ssh -o ServerAliveInterval=60 -i ~/.ssh/id_rsa -p 2222 user@host

# 带工作目录
ssh -t user@host "cd /home/user/projects && exec $SHELL -l"
```

#### 连接选项
- **保持连接**: ServerAliveInterval=60 防止连接超时
- **连接超时**: ConnectTimeout=10 避免长时间等待
- **终端分配**: -t 选项确保正确的终端环境
- **登录Shell**: exec $SHELL -l 启动登录Shell

### 🧪 连接测试

#### 测试功能
- **快速测试**: 点击"🔍 测试"按钮进行连接测试
- **批处理模式**: 使用 `ssh -o BatchMode=yes` 进行非交互测试
- **结果显示**: 显示连接成功、失败原因或超时信息

#### 测试结果类型
- ✅ **连接成功**: 服务器可达且认证成功
- ❌ **连接失败**: 网络不可达或服务器拒绝连接
- ⏰ **连接超时**: 网络延迟过高或服务器无响应
- 🔐 **认证失败**: 用户名、密码或密钥错误
- 🚫 **权限拒绝**: 用户无SSH登录权限

### 🔧 故障排除

#### 常见问题

##### 连接失败
```bash
# 检查网络连接
ping server.example.com

# 检查SSH服务
telnet server.example.com 22

# 检查防火墙设置
# 确保SSH端口（默认22）未被阻止
```

##### 认证失败
```bash
# 检查用户名和密码
ssh user@server.example.com

# 检查私钥权限
chmod 600 ~/.ssh/id_rsa

# 检查公钥是否已添加到服务器
cat ~/.ssh/id_rsa.pub
# 将公钥内容添加到服务器的 ~/.ssh/authorized_keys
```

##### SSH Agent问题
```bash
# 检查SSH Agent状态
ssh-add -l

# 重启SSH Agent
killall ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa
```

#### 日志调试
- 查看应用日志获取详细错误信息
- 使用 `ssh -v` 选项进行详细调试
- 检查服务器的SSH日志（通常在 `/var/log/auth.log`）

### 💡 最佳实践

#### 安全建议
1. **优先使用密钥认证**: 比密码认证更安全
2. **定期更换密钥**: 建议每年更换SSH密钥
3. **使用强密码**: 如果必须使用密码认证，确保密码足够复杂
4. **限制SSH访问**: 在服务器上配置SSH访问限制
5. **定期备份配置**: 导出配置文件进行备份

#### 管理建议
1. **合理命名**: 使用有意义的服务器名称
2. **标签分类**: 使用标签对服务器进行分类管理
3. **定期清理**: 删除不再使用的服务器配置
4. **团队协作**: 使用安全模式导出共享服务器列表
5. **监控使用**: 定期查看连接统计，优化常用服务器配置
