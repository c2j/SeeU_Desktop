# 语义索引重建功能使用指南

## 功能概述

语义索引重建功能允许用户在升级语义模型或需要优化搜索性能时，重新构建所有笔记的语义向量索引。这个功能特别适用于：

- 升级到新版本的语义模型后，需要应用最新的向量化技术
- 系统性能优化，清理和重建索引以提高搜索效率
- 解决索引损坏或不一致的问题

## 功能位置

在 SeeU Desktop 应用中：
1. 点击顶部导航栏的"设置"
2. 在左侧设置分类中选择"笔记设置"
3. 找到"语义搜索设置"部分

## 功能界面

### 语义搜索状态显示
- **语义搜索状态**：显示当前是否启用语义搜索
- **索引版本**：显示当前语义索引的版本号

### 操作按钮
- **🔄 重建语义索引**：重新构建所有笔记的语义向量索引
- **🗑️ 清除索引**：清除所有语义索引数据，释放存储空间

### 进度显示
- 重建过程中会显示进度条和状态信息
- 完成后会显示重建结果（成功索引的笔记数量或错误信息）

### 设置选项
- **默认使用语义搜索**：启用后，搜索时将优先使用语义搜索而非传统关键词搜索

## 使用步骤

### 重建语义索引

1. **准备工作**
   - 确保语义搜索功能已启用
   - 建议在系统空闲时进行，避免影响正常使用

2. **执行重建**
   - 点击"🔄 重建语义索引"按钮
   - 系统会显示确认对话框，说明重建过程可能需要一些时间
   - 确认后开始重建过程

3. **监控进度**
   - 重建过程中会显示进度条
   - 可以看到当前处理的笔记数量
   - 整个过程在后台进行，不会阻塞界面

4. **完成确认**
   - 重建完成后会显示成功消息
   - 索引版本号会自动更新
   - 可以立即使用新的语义搜索功能

### 清除语义索引

1. **使用场景**
   - 需要释放存储空间
   - 解决索引损坏问题
   - 临时禁用语义搜索

2. **执行清除**
   - 点击"🗑️ 清除索引"按钮
   - 系统会警告此操作不可逆
   - 确认后清除所有语义索引数据

3. **后续操作**
   - 清除后语义搜索将不可用
   - 需要重新执行"重建语义索引"来恢复功能

## 技术细节

### 重建过程
1. **清除现有索引**：删除所有现有的语义向量数据
2. **批量处理**：以10个笔记为一批进行处理，避免内存溢出
3. **向量化**：使用最新的BGE中文语义模型重新生成向量
4. **索引构建**：重新构建向量索引以优化搜索性能
5. **版本更新**：自动更新索引版本号

### 性能考虑
- **处理时间**：取决于笔记数量，大约每秒处理10-20个笔记
- **内存使用**：批量处理机制控制内存使用在合理范围内
- **存储空间**：每个笔记大约需要1.5KB的向量存储空间

### 错误处理
- 网络连接问题：会自动重试
- 内存不足：会减少批处理大小
- 模型加载失败：会显示详细错误信息
- 数据库错误：会回滚到重建前状态

## 最佳实践

### 何时重建索引
- **模型升级后**：升级BGE模型或其他语义模型后
- **性能下降时**：搜索速度明显变慢时
- **定期维护**：建议每3-6个月重建一次
- **数据迁移后**：从其他系统导入大量笔记后

### 注意事项
- **备份数据**：重建前建议备份重要笔记
- **选择时机**：在系统空闲时进行，避免影响正常工作
- **网络稳定**：确保网络连接稳定，避免中断
- **存储空间**：确保有足够的存储空间

### 故障排除
- **重建失败**：检查日志文件，通常是内存或存储空间不足
- **搜索异常**：尝试清除索引后重新重建
- **性能问题**：检查系统资源使用情况
