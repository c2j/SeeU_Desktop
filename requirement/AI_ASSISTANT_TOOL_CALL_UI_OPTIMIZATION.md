# AI助手工具调用UI优化文档

## 概述

本文档描述了对SeeU Desktop AI助手工具调用功能的UI优化，包括强制非流式请求和改进的工具调用确认界面。

## 优化内容

### 1. 强制非流式请求当发送消息带工具时

**问题：** 之前当用户设置为流式消息时，带工具的请求可能无法正确处理tool_calls响应。

**解决方案：** 修改消息发送逻辑，当检测到有工具时自动强制使用非流式请求。

**实现位置：** `crates/aiAssist/src/state.rs` 第497行

**核心逻辑：**
```rust
// 当有工具时，强制使用非流式请求
let use_streaming = ai_settings.streaming && tools.is_none();

if use_streaming {
    // 使用流式API（无工具）
    let result = api_service.send_chat_stream(...).await;
} else {
    // 使用非流式API（有工具或用户设置为非流式）
    let result = api_service.send_chat_with_tools_full_response(...).await;
}
```

**优势：**
- 确保工具调用响应能够正确解析
- 保持用户体验的一致性
- 自动处理，无需用户手动切换

2. AI助手检测到需要工具调用

## 安全考虑

1. **参数检查**：用户可以在确认前查看所有工具调用参数
2. **安全提醒**：明显的警告信息提醒用户注意安全
3. **手动确认**：所有工具调用都需要用户明确确认
4. **只读显示**：参数以只读模式显示，防止意外修改
