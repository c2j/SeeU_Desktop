# AI助手工具调用自动反馈功能说明

## 功能概述

AI助手现在支持在工具调用执行完成后，自动将工具调用结果（连同前序对话）发送给大模型，以获得大模型的完整反馈。这个功能让AI助手能够基于工具执行结果提供更智能和有用的回应。

## 功能特点

1. **自动触发**: 当用户执行工具调用并获得结果后，系统会自动发送包含工具调用结果的对话历史给大模型
2. **完整上下文**: 发送给大模型的消息包含完整的对话历史和工具调用结果
3. **智能反馈**: 大模型可以基于工具执行结果提供分析、总结或进一步的建议
4. **无缝体验**: 整个过程对用户透明，无需额外操作

## 工作流程

1. 用户发送消息给AI助手
2. AI助手识别需要调用工具并返回工具调用请求
3. 用户点击"执行"按钮执行工具调用
4. 工具执行完成，结果被添加到聊天记录中
5. **新功能**: 系统自动将包含工具调用结果的完整对话历史发送给大模型
6. 大模型基于工具执行结果生成智能反馈
7. 用户看到大模型的完整分析和建议

## 技术实现

### 核心组件

1. **自动发送机制** (`send_tool_results_to_llm`)
   - 在工具调用结果添加完成后自动触发
   - 准备包含工具调用结果的消息历史
   - 异步发送给大模型

2. **消息格式化** (`prepare_messages_for_api_with_tool_results`)
   - 将工具调用和结果转换为大模型可理解的格式
   - 保持对话上下文的完整性

3. **结果处理**
   - 创建新的助手消息接收大模型反馈
   - 支持流式输出显示

### 代码位置

- 主要实现: `src/app.rs` 中的 `send_tool_results_to_llm` 方法
- 消息准备: `crates/aiAssist/src/state.rs` 中的 `prepare_messages_for_api_with_tool_results` 方法

## 使用示例

### 场景1: 文件操作
1. 用户: "请帮我读取项目根目录下的README.md文件"
2. AI助手: 返回工具调用请求（read_file）
3. 用户: 点击执行按钮
4. 工具执行: 读取文件内容
5. **自动反馈**: AI助手基于文件内容提供项目分析和建议

### 场景2: 搜索操作
1. 用户: "搜索关于Rust编程的最新信息"
2. AI助手: 返回工具调用请求（web_search）
3. 用户: 点击执行按钮
4. 工具执行: 获取搜索结果
5. **自动反馈**: AI助手基于搜索结果提供信息总结和相关建议

## 配置要求

- 确保AI设置中配置了有效的API密钥
- 确保选择了支持工具调用的模型
- 确保网络连接正常

## 日志信息

系统会记录以下关键日志：
- `🤖 开始自动发送工具调用结果给大模型获取完整反馈`
- `📤 准备发送 X 条消息给大模型（包含工具调用结果）`
- `✅ 大模型反馈获取成功`
- `❌ 获取大模型反馈失败`

## 注意事项

1. 该功能仅在工具调用执行成功后触发
2. 如果AI设置不完整（如缺少API密钥），功能会自动跳过
3. 网络错误或API错误不会影响原有的工具调用结果显示
4. 大模型的反馈会作为新的助手消息显示在聊天界面中
