# iSearch 搜索结果菜单优化

## 🎯 优化目标

优化 iSearch 搜索结果页面的右键菜单功能，解决菜单顺序、属性功能和复制功能的问题，提升用户体验和功能实用性。

## 🐛 问题分析

### 原始问题
1. **菜单顺序不合理** - "复制"菜单在"属性"之前，不符合常见的UI设计习惯
2. **属性功能无响应** - 点击"属性"只是打印日志，没有实际的用户界面反馈
3. **复制功能重复** - "路径"和"路径+名称"返回相同的内容，功能重复

### 用户期望
- **菜单顺序**: "属性"应该在"复制"之前，符合文件管理器的常见习惯
- **属性功能**: 点击属性应该显示详细的文件信息对话框
- **复制功能**: "路径"应该只复制目录路径，"路径+名称"复制完整路径

## ✅ 优化方案

### 1. 🔄 菜单顺序调整

#### 原始顺序
```
... 菜单
├── 📄 复制
│   ├── 名称
│   ├── 路径
│   └── 路径+名称
├── ─────────
└── 属性
```

#### 优化后顺序
```
... 菜单
├── 📋 属性
├── ─────────
└── 📄 复制
    ├── 名称
    ├── 路径
    └── 路径+名称
```

### 2. 📋 属性功能完整实现

#### 新增状态管理
```rust
// 文件属性对话框状态
pub show_properties_dialog: bool,
pub properties_file: Option<SearchResult>,
```

#### 属性对话框设计
- **文件图标和名称** - 根据文件类型显示相应图标
- **详细信息网格** - 名称、路径、类型、大小、修改时间、搜索评分
- **内容预览** - 如果有内容预览则显示（带滚动条）
- **操作按钮** - 打开文件夹、打开文件、关闭对话框

### 3. 📄 复制功能区分

#### 修复前的问题
```rust
// 路径和路径+名称返回相同内容
pub fn copy_filepath(&self, result: &SearchResult) {
    self.copy_to_clipboard(&result.path); // 完整路径
}

pub fn copy_path_and_name(&self, result: &SearchResult) {
    let combined = format!("{}", result.path); // 也是完整路径
    self.copy_to_clipboard(&combined);
}
```

#### 修复后的实现
```rust
// 路径 - 只复制目录路径
pub fn copy_filepath(&self, result: &SearchResult) {
    let path = std::path::Path::new(&result.path);
    if let Some(parent) = path.parent() {
        self.copy_to_clipboard(&parent.to_string_lossy());
    } else {
        self.copy_to_clipboard(&result.path);
    }
}

// 路径+名称 - 复制完整路径
pub fn copy_path_and_name(&self, result: &SearchResult) {
    self.copy_to_clipboard(&result.path);
}
```
