# iSearch 滚动条修复 - 解决"高级搜索语法帮助"遮挡状态栏问题

## 🐛 问题描述

在之前的布局修复中，虽然解决了搜索结果覆盖状态栏的问题，但当用户展开"高级搜索语法帮助"可折叠区域时，增加的内容仍然会遮挡状态栏。这是因为可折叠内容没有被包含在滚动区域内。

## 🔍 问题分析

### 问题场景
1. **正常状态** - iSearch 工作区高度适合，状态栏可见 ✅
2. **展开帮助** - "高级搜索语法帮助"展开后，内容增加导致再次遮挡状态栏 ❌

### 根本原因
- **局部滚动限制**: 只有搜索结果区域有滚动条，其他内容区域没有滚动保护
- **可折叠内容未受限**: "高级搜索语法帮助"等可折叠内容不在滚动区域内
- **内容动态变化**: 可折叠区域的展开/收起会动态改变内容高度

## ✅ 解决方案

### 核心策略：为整个中央面板添加滚动区域

通过在 `CentralPanel` 内部添加一个包含所有内容的 `ScrollArea`，确保任何内容超出可用高度时都会显示滚动条，而不是遮挡状态栏。

### 实现方案

#### 1. 外层滚动区域包装
```rust
egui::CentralPanel::default().show_inside(ui, |ui| {
    // 为整个中央面板内容添加滚动区域
    let central_height = ui.available_height();
    egui::ScrollArea::vertical()
        .max_height(central_height)
        .auto_shrink([false, true])
        .show(ui, |ui| {
            // 所有中央面板内容都在这个滚动区域内
            ui.heading("搜索结果");
            
            // 无索引目录状态
            // 空搜索状态 + 高级搜索语法帮助 ← 现在受滚动保护
            // 搜索中状态
            // 搜索结果 (内层还有自己的滚动区域)
        });
});
```

#### 2. 双层滚动保护机制
- **外层保护**: 整个中央面板内容的滚动区域
- **内层保护**: 搜索结果的专用滚动区域 (保持原有功能)

#### 3. 高度计算优化
```rust
// 外层滚动区域 - 使用全部可用高度
let central_height = ui.available_height();

// 内层滚动区域 - 调整预留空间
let remaining_height = ui.available_height() - 80.0; // 减少预留空间
egui::ScrollArea::vertical()
    .max_height(remaining_height.max(200.0)) // 提高最小高度
```

## 🔧 技术实现细节

### 修改的代码结构

#### 原始结构 (有问题)
```
CentralPanel
├── heading("搜索结果")
├── 无索引目录状态
├── 空搜索状态
│   └── collapsing("高级搜索语法帮助") ← 无滚动保护
├── 无结果状态
├── 搜索中状态
└── 搜索结果
    └── ScrollArea (内层滚动) ← 仅此处有滚动
```

#### 修复后结构 (正确)
```
CentralPanel
└── ScrollArea (外层滚动) ← 新增，保护所有内容
    ├── heading("搜索结果")
    ├── 无索引目录状态
    ├── 空搜索状态
    │   └── collapsing("高级搜索语法帮助") ← 现在受保护
    ├── 无结果状态
    ├── 搜索中状态
    └── 搜索结果
        └── ScrollArea (内层滚动) ← 保持原有功能
```
