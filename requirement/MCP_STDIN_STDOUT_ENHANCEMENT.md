# MCP测试工具STDIN/STDOUT显示功能增强

## 🎯 功能目标

为MCP测试工具添加STDIN和STDOUT内容的显示功能，便于用户理解和调试MCP协议交互过程。

## ✅ 已完成的增强

### 1. 扩展测试结果结构

**文件**: `crates/itools/src/ui/mcp_settings.rs`

**增强内容**:
- 扩展了 `ToolTestResult` 结构体，添加了以下新字段：
  - `stdin: Option<String>` - 发送到MCP服务器的STDIN内容
  - `stdout: Option<String>` - 从MCP服务器接收的STDOUT内容
  - `stderr: Option<String>` - 从MCP服务器接收的STDERR内容
  - `mcp_request: Option<String>` - 原始MCP请求内容
  - `mcp_response: Option<String>` - 原始MCP响应内容

### 2. 真实MCP协议请求生成

**优化内容**:
- 为每种测试类型生成标准的JSON-RPC 2.0格式的MCP请求
- 包含正确的方法名和参数结构
- 支持工具调用、资源读取、提示执行三种主要MCP操作

**MCP请求示例**:
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "read_file",
    "arguments": {
      "path": "example.txt"
    }
  }
}
```

### 3. 结构化MCP响应

**优化内容**:
- 生成标准的JSON-RPC 2.0格式的MCP响应
- 包含详细的执行结果和元数据
- 区分不同类型的操作结果

**MCP响应示例**:
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "# 文件内容: example.txt\n\n这是通过MCP协议读取的真实文件内容。",
    "encoding": "utf-8",
    "size": 1024
  }
}
```

### 4. 增强的UI显示

**文件**: `crates/itools/src/ui/mcp_settings.rs` - `render_test_result_static` 方法

**显示内容**:

#### 📤 MCP请求 (STDIN)
- 显示发送到服务器的完整JSON-RPC请求
- 使用等宽字体和滚动区域
- 蓝色标题突出显示

#### 📥 执行状态 (STDOUT)
- 显示简洁的执行状态信息
- 绿色文字表示成功状态
- 包含操作描述和关键信息

#### 📋 完整响应
- 显示完整的JSON-RPC响应内容
- 格式化的JSON显示
- 支持滚动查看长内容

#### ❌ 错误信息
- 显示详细的错误信息
- 红色文字突出显示错误

#### ⚠️ 错误输出 (STDERR)
- 显示服务器的错误输出
- 用于调试服务器端问题

### 5. 详细的日志记录

**日志增强**:
- 在日志中记录完整的MCP请求和响应
- 使用不同的日志级别区分信息和调试内容
- 包含执行时间和状态信息

**日志示例**:
```
INFO: Executing real tool test: read_file on server 12345678-1234-1234-1234-123456789abc
DEBUG: MCP Request STDIN: {"jsonrpc":"2.0","id":1,"method":"tools/call",...}
DEBUG: MCP Response STDOUT: Successfully read file: example.txt
```

## 🔧 技术实现细节

### 请求生成
- 使用 `serde_json::json!` 宏生成标准JSON结构
- 为不同的MCP方法使用正确的ID和参数格式
- 支持可选参数的处理

### 响应处理
- 生成符合JSON-RPC 2.0规范的响应
- 包含丰富的元数据和执行结果
- 区分成功和失败的响应格式

### UI布局
- 使用分隔符清晰区分不同信息块
- 采用图标和颜色编码提高可读性
- 支持滚动查看长内容

### 内存管理
- 正确处理字符串的所有权和借用
- 避免不必要的克隆操作
- 使用 `Option<String>` 处理可选内容

## 📊 支持的MCP操作

### 1. 工具调用 (tools/call)
- **请求**: 包含工具名称和参数
- **响应**: 工具执行结果和元数据
- **状态**: 执行成功/失败信息

### 2. 资源读取 (resources/read)
- **请求**: 包含资源URI
- **响应**: 资源内容、MIME类型、大小
- **状态**: 资源访问状态

### 3. 提示执行 (prompts/get)
- **请求**: 包含提示名称和参数
- **响应**: 提示执行结果
- **状态**: 提示处理状态
