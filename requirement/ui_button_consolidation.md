# UI按钮合并优化

## 问题描述

在SeeU Desktop的搜索设置界面中存在重复和混淆的按钮：

### 1. 重复的索引按钮
- **"重新索引所有目录"** - 在搜索选项区域
- **"更新全部索引"** - 在目录管理区域

### 2. 重复的移除目录按钮
- **"- 移除目录"** - 在添加目录按钮旁边
- **"➖ 移除目录"** - 在目录管理按钮区域

## 功能分析

### 索引功能差异

#### `reindex_all_directories()`
- **功能**: 完全重建索引
- **过程**: 先清空所有索引，然后重新索引所有目录
- **用途**: 应用代码更新后的功能改进，解决索引结构变化问题
- **影响**: 会清除搜索缓存，重建整个索引

#### `update_all_indexes()`
- **功能**: 更新现有索引
- **过程**: 在现有索引基础上重新索引目录内容
- **用途**: 日常的索引更新，同步文件变化
- **影响**: 保留现有索引结构，增量更新

### 用户体验问题

1. **功能混淆**: 用户难以理解两个索引按钮的区别
2. **界面冗余**: 重复的按钮增加界面复杂度
3. **操作困惑**: 不知道应该使用哪个按钮

## 优化方案

### 1. 索引按钮合并

**合并策略**:
- 保留`reindex_all_directories()`功能
- 统一按钮文本为"🔄 重新索引全部"
- 统一提示文本为"重新索引所有目录，应用最新功能改进"

**理由**:
- 完全重建索引包含了更新索引的功能
- 对用户来说，"重新索引"比"更新索引"更容易理解
- 避免用户在两个相似功能间犹豫

### 2. 移除目录按钮合并

**合并策略**:
- 保留在目录管理主按钮区域的移除功能
- 移除重复的移除按钮
- 统一使用"- 移除目录"样式

**理由**:
- 将相关功能集中在一个区域
- 减少界面元素，提升简洁性
- 保持功能的逻辑分组

## 实现细节

### 1. 设置界面优化

#### 目录管理主按钮区域
```rust
// Directory management buttons
ui.horizontal(|ui| {
    if ui.button("+ 添加目录").clicked() {
        app.isearch_state.open_directory_dialog();
    }

    // Remove directory button (only enabled if a directory is selected)
    if let Some(selected) = app.isearch_state.selected_directory {
        if ui.button("- 移除目录").clicked() {
            app.isearch_state.remove_directory(selected);
        }
    } else {
        ui.add_enabled(false, egui::Button::new("- 移除目录"));
    }

    // Unified reindex button
    if ui.button("🔄 重新索引全部").on_hover_text("重新索引所有目录，应用最新功能改进").clicked() {
        app.isearch_state.reindex_all_directories();
    }
});
```

#### 移除重复按钮
- 删除目录列表下方的重复"移除目录"按钮
- 删除搜索选项区域的重复"重新索引"按钮

### 2. 搜索界面同步

#### 侧边栏索引按钮
```rust
// Reindex all directories button
if ui.button("🔄 重新索引全部").on_hover_text("重新索引所有目录，应用最新功能改进").clicked() {
    state.reindex_all_directories();
}
```

### 3. 状态提示优化

#### 更新提示文本
```rust
ui.with_layout(egui::Layout::right_to_left(egui::Align::Center), |ui| {
    ui.label(egui::RichText::new("选择目录后可移除或重新索引").weak());
});
```
