### 阶段2：配置管理测试

#### 2.1 检查配置文件
```bash
# 查看配置目录
ls -la ~/Library/Application\ Support/SeeU_Desktop/semantic_search/
# 或 Linux: ~/.local/share/SeeU_Desktop/semantic_search/
# 或 Windows: %APPDATA%\SeeU_Desktop\semantic_search\

# 查看配置文件
cat ~/Library/Application\ Support/SeeU_Desktop/semantic_search/config.toml
```

**预期结果**：
- 配置目录存在
- 包含默认配置文件

#### 2.2 修改配置启用语义搜索
创建或编辑配置文件：
```toml
# config.toml
enabled = true

[helix_config]
database_path = ""  # 留空使用默认路径
port = 6969
connection_timeout = 30
query_timeout = 30

[embedding_config]
provider = "openai"  # 或 "local"
api_key = "your-openai-api-key"  # 如果使用OpenAI
model = "text-embedding-3-small"
batch_size = 10
cache_size = 1000

[search_weights]
semantic_weight = 0.7
keyword_weight = 0.3
```

#### 2.3 重启应用验证配置
```bash
# 重启应用
cargo run

# 观察日志，应该看到：
# ✅ "语义搜索已启用，开始初始化..."
# ✅ "✅ 语义搜索模块初始化成功"
```

## 🔍 故障排除

### 常见问题

#### 1. 语义搜索初始化失败
**可能原因**：
- HelixDB未安装或不在PATH中
- 端口6969被占用
- 配置文件错误

**解决方案**：
```bash
# 检查HelixDB安装
which helix

# 检查端口占用
lsof -i :6969

# 查看详细错误日志
RUST_LOG=debug cargo run
```

#### 2. 向量化失败
**可能原因**：
- API密钥无效
- 网络连接问题
- 本地模型服务未启动

**解决方案**：
```bash
# 验证API密钥
curl -H "Authorization: Bearer your-api-key" \
     https://api.openai.com/v1/models

# 检查本地模型服务
curl http://localhost:11434/api/tags  # Ollama
```

#### 3. 搜索无结果
**可能原因**：
- 笔记未完成向量化
- 搜索权重配置问题
- 查询词过于具体

**解决方案**：
- 等待向量化完成
- 调整搜索权重
- 尝试更通用的查询词
