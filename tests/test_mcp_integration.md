# MCP工具能力集成测试指南

## 测试目标
验证AI助手能否正确地将选中的MCP服务器的工具能力通过LLM请求的tools参数发送到大模型。

## 测试前准备

### 1. 确保有可用的MCP服务器
- 在iTools中添加并测试一个MCP服务器（如CounterTool）
- 确保服务器状态为绿灯（已测试通过）
- 确保服务器有工具定义

### 2. 配置AI助手
- 确保AI助手已配置有效的API设置
- 建议使用支持Function Calling的模型（如GPT-4、Claude等）

## 测试步骤

### 步骤1：验证MCP服务器选择日志
1. 打开AI助手侧边栏
2. 在MCP下拉框中选择一个绿灯服务器
3. **预期日志**：
   ```
   🎯 AI助手 - 用户选择MCP服务器: [服务器名称]
   📋 服务器能力: 工具:X 资源:Y 提示:Z
   ```

### 步骤2：验证工具定义转换日志
1. 在选中MCP服务器的情况下，发送一条消息（如"帮我计数"）
2. **预期日志**：
   ```
   🔧 AI助手 - 选中MCP服务器: [服务器名称]
   📋 MCP服务器能力统计:
     - 工具数量: X
     - 资源数量: Y
     - 提示数量: Z
   🛠️ 可用工具列表:
     1. [工具名称] - [工具描述]
   ✅ 已将 X 个MCP工具转换为OpenAI Function Calling格式
   ```

### 步骤3：验证API请求日志
1. 继续观察发送消息后的日志
2. **预期日志**：
   ```
   📤 发送请求到: [API URL]
   🤖 使用模型: [模型名称]
   💬 消息数量: X
   🛠️ 发送工具定义给LLM:
     - 工具数量: X
     1. 工具名称: [工具名称]
        工具描述: [工具描述]
        参数数量: X
          - [参数名称]
   ```

### 步骤4：验证LLM响应处理
1. 如果LLM返回工具调用，应该看到：
   ```
   🔄 MCP集成管理器 - 处理工具调用响应:
     - 工具调用数量: X
     1. 解析工具调用: [工具名称]
        ✅ 成功解析为MCP工具调用
        - 原始名称: [工具名称]
        - 调用类型: CallTool
        - 目标服务器: [服务器名称] ([服务器ID])
   📦 创建工具调用批次:
     - 批次ID: [UUID]
     - 待处理工具调用数量: X
   ✅ 工具调用批次已创建，等待用户确认
   ```

## 测试验证点

### ✅ 成功指标
- [ ] MCP服务器选择被正确记录
- [ ] 工具定义被正确转换为OpenAI格式
- [ ] API请求包含tools参数
- [ ] 工具调用响应被正确解析
- [ ] 显示工具调用确认对话框

### ❌ 失败指标
- [ ] 选择MCP服务器后没有日志输出
- [ ] 发送消息时显示"🚫 未发送工具定义 (无选中的MCP服务器)"
- [ ] API请求中没有tools参数
- [ ] LLM返回工具调用但没有被处理

## 常见问题排查

### 问题1：MCP服务器不显示在下拉框中
- 检查服务器是否为绿灯状态
- 检查服务器是否有能力信息
- 尝试点击刷新按钮

### 问题2：选择服务器后没有工具定义
- 检查服务器能力是否包含工具
- 检查工具定义的格式是否正确

### 问题3：API请求没有包含tools参数
- 检查是否正确选择了MCP服务器
- 检查工具转换逻辑是否正常

```

### 测试用例3：无工具调用

```

## 调试技巧

1. **启用详细日志**：确保日志级别设置为INFO或DEBUG
2. **检查网络请求**：可以使用网络抓包工具查看实际的API请求
3. **验证JSON格式**：检查工具定义的JSON格式是否符合OpenAI规范
4. **测试不同模型**：某些模型可能对Function Calling支持更好

## 预期结果

如果一切正常，您应该能够：
1. 在AI助手中选择MCP服务器
2. 发送消息时看到工具定义被发送给LLM
3. LLM返回工具调用时看到确认对话框
4. 确认后执行工具调用并获得结果

这验证了MCP工具能力已成功集成到AI助手的Function Calling流程中。
