# 思源笔记导入工具快速开始指南

## 前置要求

1. 确定思源笔记的工作空间目录位置
2. 选择目标笔记软件（如Obsidian、Notion、Logseq等）
3. 准备开发环境（推荐Go或Python）

## 快速实现步骤

### 第一步：识别笔记本

```bash
# 工作空间目录结构示例
/Users/username/SiYuan/
├── 20210808180117-czj9bvb/    # 笔记本1
├── 20210808180117-6v0mkxr/    # 笔记本2
├── assets/
└── .siyuan/
```

### 第二步：读取笔记本配置

```json
// .siyuan/conf.json
{
  "name": "我的笔记本",
  "icon": "1f4d4",
  "closed": false
}
```

### 第三步：解析.sy文档文件

每个.sy文件都是一个JSON文档：
```json
{
  "ID": "20200812220555-lj3enxa",
  "Properties": {
    "title": "文档标题",
    "updated": "20241125224159"
  },
  "Children": [...]
}
```

### 第四步：递归处理节点

主要节点类型处理：

1. **NodeText** - 纯文本
2. **NodeParagraph** - 段落
3. **NodeHeading** - 标题（1-6级）
4. **NodeList** - 列表
5. **NodeCodeBlock** - 代码块
6. **NodeBlockquote** - 引用块

### 第五步：处理资源文件

```go
// 资源文件通常在以下位置：
// 1. 全局assets目录
// 2. 笔记本内的assets目录
// 3. 文档中的相对路径引用
```

## 常见转换场景

### 转换为Markdown

```python
def to_markdown(node):
    if node['Type'] == 'NodeHeading':
        level = node.get('HeadingLevel', 1)
        text = extract_text(node)
        return f"{'#' * level} {text}\n"
    elif node['Type'] == 'NodeParagraph':
        return extract_text(node) + '\n\n'
    # ... 其他类型处理
```

### 转换为Obsidian

特殊处理：
- 块引用 → `[[文档名#块]]`
- 标签 → `#标签名`
- 双链 → `[[文档名]]`

### 转换为Notion

特殊处理：
- 数据库视图 → Notion Database
- 超级块 → Notion Columns
- 属性 → Notion Properties

## 实用工具函数

### 文本提取
```go
func extractText(node Node) string {
    if node.Type == "NodeText" {
        return node.Data
    }
    var result strings.Builder
    for _, child := range node.Children {
        result.WriteString(extractText(child))
    }
    return result.String()
}
```

## 测试建议

1. **从简单文档开始**：选择只包含文本和标题的文档进行测试
2. **逐步增加复杂度**：添加列表、代码块、图片等
3. **验证引用关系**：确保块引用和双链正确转换
4. **检查资源文件**：验证图片和附件是否正确复制

## 常见问题解决

### Q: 中文文件名乱码
A: 确保使用UTF-8编码读取文件

### Q: 块引用无法解析
A: 需要先建立ID到文档的映射表

### Q: 图片路径错误
A: 检查相对路径计算和资源文件复制逻辑

### Q: 嵌套列表格式错误
A: 递归处理NodeListItem时注意层级关系

## 性能优化建议

1. **批量处理**：一次性读取多个文档
2. **并发处理**：使用goroutine或多线程
3. **内存管理**：大文档分块处理
4. **缓存机制**：缓存已解析的文档结构

## 扩展功能

- **增量同步**：基于更新时间判断
- **冲突处理**：处理同名文档
- **格式验证**：检查转换结果的正确性
- **进度显示**：添加进度条和日志
